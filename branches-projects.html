Branches & Projects

Use branches for focused changes and a project board to organize what’s next.

How branches work

A branch is basically your safe playground. Do your edits there while main stays clean and functional. Choose intent-revealing names like readme-update or feature/contact-form. Create the branch, switch to it in your IDE, and make incremental commits with descriptive messages. After testing, push the branch and open a pull request to propose merging into main.

Set up a Project

Projects help you track to-dos, bugs, and priorities. Boards show status at a glance.

  1. Go to the Projects tab (repo or profile) → New project (Board or Table).
  2. Add a Status field or columns like To do, In progress, Done.
  3. Add existing Issues or write draft notes; drag cards to update status.
  4. Use labels, assignees, and milestones to show ownership and timing.
  5. Create views/filters (e.g., “My tasks” or “This week”) to focus the board.

Break work up into small tasks, link related issues/PRs, and update the board as you go.
