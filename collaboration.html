Collaboration

Propose changes with pull requests and track work with issues so your team stays in sync.

Pull Requests (PRs)

A PR bundles your branch’s commits, shows the diff, lets people comment on lines, and can run checks before merge.

Open and merge a PR

  1. Push your branch, then click Compare & pull request.
  2. Write a clear title and short description (what changed and why).
  3. Request reviewers; address feedback with follow-up commits from your IDE.
  4. When checks pass and it’s approved, click Merge and delete the branch if finished.

Why teams use PRs: review, visibility, automated checks, and a permanent record of decisions.

Issues

Issues capture tasks, bugs, ideas, and link directly to code and PRs.

Use Issues effectively

Give the issue a clear title. In the description, state the goal of the issue; how will we know it’s finished?. Add labels (e.g., bug) so it’s easy to filter and add a milestone if there’s a due date. When the related PR is ready, include which issue it resolves: for example, Fixes #123. After that PR merges to main, the issue closes automatically and the cross-references are recorded.

Team habits that help

Use descriptive branch names (like feature/login) so reviewers immediately know the goal. Keep PRs small and focused by dropping in a screenshot or a tiny checklist to show what changed. Keep the README and project board updated. Maintain a clean README and an honest project board so the repo works as a solid portfolio piece.
