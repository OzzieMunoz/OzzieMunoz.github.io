Getting Started

Create a repo, add files, and learn a few shortcuts.

Create a repository

  1. Click New from the GitHub header or your profile.
  2. Name the repo (e.g., hello-world) and choose Public or Private.
  3. Click Create repository. In Settings, add a short description and topics.
  4. Add a README.md explaining what the project does and how to run it.

Work from your IDE

  1. On your repo page, click Code → choose Open with GitHub Desktop (or your IDE of choice) to clone locally.
  2. Open the folder in your IDE. Make small changes and save.
  3. Use your IDE or GitHub Desktop to stage and commit with a short message (e.g., “Add README”).
  4. Push to GitHub so the repo updates online.

Tip: you can also add files via the web UI (Add file → Upload files) whenever that’s quicker.

First commit tips

Make the first commit small. Add the README, maybe a starter file and write a short message like “Add README.” If you want people to reuse your code, drop in a LICENSE. Keep polishing the README as the project grows; future you (and recruiters) will thank you.

Git Commands

Even when working from an IDE, knowing the underlying Git commands is useful. The commands below cover the core workflow—clone → change → add → commit → push— plus branch and history tools. Use this table as a quick reference.

Some of the most important/common commands are:

CommandWhat it does
git clone [url] Make a local copy of an existing GitHub repository.
git status Shows the current branch and which files are modified, staged, or untracked.
git branch Lists your local branches. Use git branch [name] to create one, or git branch --all to see both local and remote-tracking branches.
git checkout [branch-name] Switch your working directory to the named branch.
git add [file] Stages the file—queues its changes so they’re included in the next commit.
git commit -m "descriptive message" Records the staged changes as a snapshot in history with a message explaining the update.
git pull Updates your current branch with commits from the remote branch (essentially fetch followed by merge).
git push Publishes your local commits to the repository’s remote.
git log Lets you scroll through commit history and messages.
git remote -v Shows the configured remotes and their URLs (e.g., the origin remote).
